# Nimra Wani | Portfolio Website 🌐

A clean, responsive, and highly interactive dual-mode portfolio website showcasing computer science foundations, full-stack engineering, and AI/ML projects. 

**Live Demo:** [https://nimrawani.vercel.app/](https://nimrawani.vercel.app/)

---

## 🌟 Visual Modes

This portfolio offers four distinct interactive experiences for visitors:
* **Classic Professional View:** A sleek, glassmorphic dark-themed developer hub featuring floating Dock navigation and smooth micro-animations.
* **Interactive Kitchen Portfolio (`?kitchen`):** An immersive cottage-core mini-game where visitors can explore projects, skills, and achievements by "cooking" recipes with specific tech stack ingredients!
* **City Quest Driving Game (`?cityquest`):** A stylized 2.5D open-world arcade driving game where you customize a vehicle (sports car, lightcycle, scooter, hoverpod) in a showroom garage and drive around a neon city to explore achievements, certifications, events, and milestones!
* **Bug Dungeon Debugging Game (`?bugdungeon`):** An atmospheric 2D retro-futuristic mechanical underground dungeon where you solve interactive coding logic challenges (CPU infinite loops, API timeouts, CSS overlaps, Git conflicts) to unlock real development stories, logs, and lessons learned!

## 🎮 Interactive Arcade Games

### 🍳 GAME 1 — NIMRA’S KITCHEN

Nimra’s Kitchen is the first and central game inside Nimra’s Arcade and represents projects, technical stacks, creativity, and the complete process of building applications. Instead of displaying projects as traditional portfolio cards, this game transforms every project into a playable cooking experience where the user learns about technologies, features, and development processes by physically preparing dishes inside an interactive kitchen environment. The concept of the game is based on the idea that every project is like a recipe made from different ingredients, where technologies such as React, TypeScript, SQL, Firebase, Docker, APIs, and deployment tools become cooking ingredients used to create the final dish. The purpose of the game is not competition or difficulty, but immersion and interaction. The user should feel like they are inside a real cooking simulator while unknowingly exploring your technical work and development journey.

When the player enters Nimra’s Kitchen from the Arcade Hub, the transition begins with the kitchen door slowly opening while warm light spills into the hallway. The camera smoothly moves inside the kitchen as ambient cooking sounds become louder. The environment immediately feels alive through small atmospheric details such as stove flames flickering softly, steam rising from simmering pots, utensils hanging near counters, reflections on polished surfaces, and subtle movement from curtains near the windows. The kitchen should feel like a believable modern cooking game environment rather than a fantasy or overly abstract space. The art style combines stylized realism with polished game aesthetics, similar to modern mobile cooking simulators or cozy indie cooking games. The environment uses warm lighting, soft shadows, natural textures, and smooth rounded shapes to create a visually comfortable and inviting atmosphere.

The kitchen itself is divided into multiple functional gameplay areas. At the center of the environment is the Main Preparation Counter, which acts as the core gameplay station where recipes are viewed, ingredients are mixed, bowls are placed, and dishes are assembled. This counter remains visually important throughout the game because it acts as the main interaction hub. Surrounding the kitchen are organized ingredient shelves and storage sections that naturally represent different categories of technologies. Frontend technologies may appear on baking shelves as flour, sugar, or packaged ingredients, while backend technologies and databases may appear inside refrigerators, sauce racks, spice cabinets, or storage jars. Development tools and deployment systems can appear as oils, sauces, seasonings, or cooking utilities. This organization helps users subconsciously understand how technologies combine together inside projects without explicitly teaching them through text.

One side of the kitchen contains the Stove Section, filled with burners, ovens, pans, pots, cooking tools, and preparation equipment. This area becomes active during gameplay and includes animated flames, bubbling oil, boiling liquids, and reactive lighting effects that respond dynamically during cooking interactions. Another important part of the environment is the Recipe Wall mounted naturally on the kitchen wall like a real restaurant order board. Instead of floating project cards or random menus, this wall displays the available project dishes in a believable and organized way. Each project appears as a dish on the menu, such as “AI Study Assistant Ramen,” “Portfolio Burger,” “IoT Cake,” or “Expense Tracker Pasta.” This instantly communicates the concept visually while maintaining immersion within the cooking theme.

The gameplay begins when the user approaches the Recipe Wall and selects one of the available dishes. Once selected, the kitchen reacts dynamically. The chosen recipe lights up, the preparation counter activates, and a cookbook opens naturally onto the counter with a smooth page-turning animation. The recipe interface is designed to resemble a real cookbook rather than a technical dashboard. It includes handwritten notes, ingredient checklists, cooking steps, icons, and ingredient quantities presented in a visually playful yet organized manner. Technologies are transformed into believable ingredient equivalents such as “React Flour,” “TypeScript Pepper,” “Firebase Syrup,” or “SQL Sauce.” The recipe may display instructions such as “Add two cups of React Flour, one spoon of SQL Sauce, and a pinch of TypeScript Pepper.” This creates a memorable metaphor that makes the tech stack visually understandable while still feeling like a real cooking game.

After reading the recipe, the player begins gathering ingredients from around the kitchen. Movement and interactions should feel smooth and tactile. When hovering over ingredients, subtle animations occur such as soft bouncing, gentle glowing outlines, or slight tilts that provide visual feedback without becoming distracting. Dragging ingredients should feel satisfying through smooth cursor tracking, light weight physics, and accompanying sound effects. When ingredients are placed into bowls, pans, or mixers, the game responds with realistic animations such as powder puffs, liquid pours, splashes, steam effects, or bubbling reactions depending on the ingredient type. These microinteractions are extremely important because they create immersion and make the gameplay feel polished and professional.

Cooking itself becomes the core gameplay loop. The player may need to mix ingredients inside bowls, fry items on the stove, blend sauces, heat pans, bake dishes in ovens, or plate final meals using simple interactive mechanics. None of these mechanics should feel stressful or overly complicated. The experience should remain smooth, enjoyable, and intuitive so that the focus stays on exploration and creativity rather than challenge. Small animations throughout the process add realism and satisfaction, such as rotating knives during chopping actions, spinning spoons during mixing, flickering stove flames, naturally fading steam, bubbling sauces, clinking utensils, and subtle kitchen vibrations during active cooking sequences. Every action should generate feedback through movement, sound, and visual response.

The scoring system inside Nimra’s Kitchen is intentionally designed to avoid ranking projects against one another. There are no difficulty levels because every project is treated as equally valuable and important. Instead, the star system only measures how accurately the user followed the recipe. If all ingredients are selected correctly, the player receives a perfect five-star result. If one incorrect ingredient is used, half a star is deducted, resulting in a 4.5-star score. If two ingredients are incorrect, the player receives four stars. Mistakes never lead to failure or punishment. Instead, they trigger playful and lighthearted feedback such as tiny smoke puffs, slight overcooking effects, chef reactions, or subtle cracking animations on stars. This creates engagement without making the experience frustrating or competitive.

One of the most important moments in the entire game is the final transition from cooking to project showcase. Once the dish is fully prepared and plated beautifully, the camera slowly zooms toward the meal while cinematic lighting enhances the presentation. Steam rises naturally from the food and gradually fills the screen. As the steam clears, the scene transitions seamlessly into the actual project showcase page. The food visually transforms into the project presentation itself, where screenshots, live demos, GitHub repositories, features, animations, architecture details, and development notes appear naturally inside the interface. This transition is designed to create a memorable “wow” moment that emotionally connects gameplay with portfolio exploration.

The UI and UX design of Nimra’s Kitchen focuses heavily on warmth, readability, tactility, and smooth interaction design. The interface uses rounded edges, soft shadows, clean typography, organized layouts, and polished animations inspired by modern mobile cooking games. UI elements never overcrowd the screen and always maintain strong visual hierarchy. Important information such as ingredients, stars, recipes, and cooking steps remains easy to understand at all times. Every button, hover state, animation, and interaction is designed to provide immediate feedback so the user constantly feels connected to the environment. The navigation system remains subtle but accessible, allowing users to return to the Arcade Hub, access settings, mute sound, or switch projects without disrupting immersion.

Sound design is one of the strongest emotional components of the game. The background ambience includes subtle kitchen sounds such as distant chopping, soft sizzling, utensil movement, refrigerator hums, oven ticking, and calm upbeat music playing underneath the environment. Every interaction produces satisfying audio feedback including whisk mixing sounds, liquid pouring, chopping vegetables, oven dings, spoon stirring, bubbling soups, and plate placement sounds. These sounds make the kitchen feel responsive and alive. During successful recipe completion, soft celebratory sounds, sparkle effects, and star animations provide emotional satisfaction without becoming overly dramatic.

### 🚗 GAME 2 — CITY QUEST

City Quest acts as the achievements, certificates, awards, and milestone section of the portfolio. Instead of presenting certifications and accomplishments through static cards or timelines, this game transforms the entire career journey into an explorable city where every road, district, building, and checkpoint represents a different achievement or stage of growth. The goal of the game is exploration and discovery. The user enters a living city environment and gradually uncovers certifications, hackathons, awards, leadership roles, and important milestones by physically traveling through the world in a vehicle of choice from a showroom garage.

---

### 🧩 GAME 3 — BUG DUNGEON

Bug Dungeon represents problem-solving, debugging, technical challenges, failures, learning experiences, and the ability to think like a real developer under pressure. Unlike the Kitchen Game, which focuses on building projects, or City Quest, which focuses on achievements, Bug Dungeon is about the hidden side of development — the moments where systems fail, errors appear unexpectedly, and solutions must be discovered through logic and persistence.

## 🚀 Getting Started

To run this portfolio locally:

1. **Clone the repository:**
   ```bash
   git clone https://github.com/nimrawani04/nimrawani04.github.io.git
   ```

2. **Navigate to the directory:**
   ```bash
   cd nimrawani04.github.io
   ```

3. **Install dependencies:**
   ```bash
   npm install
   ```

4. **Run in development mode:**
   ```bash
   npm run dev
   ```

5. **Open in your browser:**
   Open [http://localhost:5173](http://localhost:5173). Click the **"Kitchen"** icon in the floating dock or add `?kitchen` to the URL to test the cooking game!
